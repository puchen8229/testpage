两年落地路线图 · 系统性进阶

基于你已有的 switcher_AItutor 代码基础，从“规则驱动”演进到“认知型 Agent”。

阶段一：框架重构 · 掌握认知 Agent 范式 第1–2个月

将原有硬编码状态机迁移到 LangGraph，让 Agent 自主规划与决策。

  • 核心任务：用 LangGraph 复刻 Socratic/Explainer 双路由，但路由决策交由 LLM 判断 (ReAct 模式)。
  • 关键学习：LangGraph 状态图、条件边、Function Calling。
  • 产出：一个能根据学生输入动态选择“引导提问”或“直接解释”的 Agent 原型。
阶段二：记忆系统 · 让 Agent 从历史中学习 第3–4个月

突破单次对话限制，建立三级记忆 (感知/短期/长期)。

  • 短期记忆：使用 ConversationSummaryBufferMemory 管理会话上下文。
  • 长期记忆：将以往成功的教学/计量分析案例存入向量库，检索相似场景策略。
  • 计量场景映射：“之前处理 DID 平行趋势不满足时是如何引导的？”
阶段三：自我反思与工具生态 第5–8个月

打造计量专属“技能包”，并引入反思机制提升可靠性。

  • 工具封装：将 statsmodels 诊断、Hausman 检验、DID 平行趋势检验封装为可调用函数。
  • 反思循环 (Reflexion)：让 Agent 在执行分析后自评：“工具变量是否弱？是否需要报告稳健标准误？”
  • 可观测性集成：接入 LangSmith 监控每一步推理与工具调用。
阶段四：自主研究助手 · 从指令到目标 第9–24个月

实现“帮我分析最低工资对就业的影响”级别的端到端自主分析。

  • 自动探查数据结构 → 推荐模型 (DID/SCM) → 执行诊断 → 产出稳健性报告。
  • 持续注入你四十年的计量直觉（例如识别 Staggered DID 时自动建议 CSDID）。
  • 最终形态：一位不知疲倦、严谨且具备学术品味的“AI 计量副驾驶”。

进阶提示：你当前的 socratic_tester.py 正是很好的评估起点；后续可将测试用例拓展到计量诊断场景。

我的补充

这些将成为 Agent 的“单元测试”，确保每次升级不退化。
